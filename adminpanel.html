1. Overview

This guide explains how to set up, configure, and host the GrabDeals Admin Panel, which allows you to manage your mobile and web apps built with Firebase and Flutter. The admin panel is developed using HTML, CSS, JavaScript, and Bootstrap, and it connects directly with your Firebase project.

2. What You'll Need

  • A Firebase account with a project already configured
  • The admin panel files (provided as a zip)
  • Access to a local server environment (XAMPP/WAMP) or shared hosting with cPanel
  • Basic file editing tool (e.g., Notepad++, VS Code)

3. How It Works

The admin panel connects directly to your Firebase Realtime Database and Authentication system. Any changes you make in the admin panel (like adding deals or editing user data) are reflected in your GrabDeals mobile app and website immediately.

6. Local Access to Admin Panel

  1. Copy the admin panel folder (e.g., grabdeals_admin) to your server root:
    • XAMPP: C:\xampp\htdocs
    • WAMP: C:\wamp64\www
  2. Rename the folder to something simple, e.g., admin.
  3. Open your browser and visit: 
    http://localhost/admin/index.html
  4. If it was named kamao_new, you'd access it via: 
    http://localhost/kamao_new/index.html
  5. To change it, just rename the folder and access accordingly. 
    http://localhost/grabdeals_admin/index.html

7. Upload to Shared Hosting (cPanel or Similar)

  1. Zip your admin panel folder.
  2. Login to your shared hosting account and go to cPanel > File Manager.
  3. Navigate to public_html/ or your preferred subdirectory.
  4. Click Upload, then select and upload your ZIP file.
  5. After upload, select the ZIP and click Extract.
  6. Your admin panel will now be available at:
    • Main directory: https://yourdomain.com/index.html
    • Subfolder: https://yourdomain.com/admin/

9. FAQ / Troubleshooting

  • Q: I see a blank screen.
    A: Check browser console for Firebase connection errors. Make sure your config is correct.
  • Q: Firebase data is not updating.
    A: Make sure correct databaseURL is set, and rules allow read/write.
  • Q: Login fails.
    A: Ensure you've created admin@admin.com user in Firebase Authentication.

10. Security Tip

After initial setup, you should change the default admin credentials inside Firebase Authentication.
Consider enabling email verification or 2FA if needed.
